Contexte du projet
Vous êtes analyste data junior dans une entreprise qui souhaite mesurer l’efficacité de ses campagnes marketing et identifier les produits les plus achetés par ses clients.

Vous disposerez d’un dataset clients incluant des informations démographiques, transactions, campagnes et produits achetés.

Objectifs du projet
Réaliser une Analyse Exploratoire des Données (EDA) pour comprendre le dataset.
Effectuer des tests statistiques afin d’analyser les relations et l’impact des campagnes sur les ventes.
Transformer et préparer les données dans Power Query pour faciliter l’analyse.
Créer une modélisation des données dans Power BI pour visualiser les campagnes et produits.
Construire un dashboard interactif et lisible focalisé sur la performance des campagnes.
Étapes principales
Étape 1 : Analyse Exploratoire des Données (Python)
Charger le dataset avec pandas et identifier les types de variables : continues vs discrètes.
Analyser les statistiques descriptives : moyenne, médiane, min, max, écart-type.
Visualiser les données :

Histogrammes et boxplots pour les ventes et montants des produits.
Bar charts pour le nombre de clients ayant accepté chaque campagne.
Vérifier les anomalies, valeurs manquantes et doublons.

Interpréter les résultats pour tirer des insights initiaux sur les clients et campagnes.

Étape 2 : Tests Statistiques
• Vérifier l’impact des campagnes sur les ventes :

t-test ou ANOVA pour comparer les montants moyens des ventes.
Chi² pour les variables catégorielles (ex. campagne acceptée vs pays).
• Explorer les corrélations entre les variables démographiques et le comportement d’achat.

• Interpréter les p-values pour identifier les facteurs significatifs.

Étape 3 : Transformation des Données (Power Query)
• Créer des tables pivot/unpivot :

Campagnes → colonne unique pour toutes les campagnes acceptées.
Produits → colonne unique pour les ventes par produit.
Plateformes → colonne unique pour le type d’achat (Web, Store, Catalogue).
• Nettoyer et renommer les colonnes pour un modèle clair et logique.

• Vérifier les relations entre les tables : ID client → campagnes, produits, plateformes.

• S’assurer que les données sont prêtes pour la modélisation et les visualisations.

Étape 4 : Modélisation des Données (Power BI)
• Définir les relations entre les tables en utilisant ID client comme clé principale.

• Vérifier la direction des filtres pour que les mesures et visualisations fonctionnent correctement.

• Créer des mesures DAX si nécessaire :

Total des ventes par campagne et par produit.
Nombre de clients ayant accepté chaque campagne.
• S’assurer que les tables pivotées communiquent correctement entre elles.

Étape 5 : Dashboard Power BI – Campaign Performance
• Créer des visualisations interactives et lisibles :

Nombre de clients par campagne.
Chiffre d’affaires par campagne et par produit.
Répartition des ventes par produit pour chaque campagne (stacked bar chart).
Répartition des achats par plateforme (web, catalogue, magasin).
• Ajouter des filtres et slicers pour faciliter l’analyse dynamique.

• Le dashboard doit être clair, intuitif et orienté business, permettant de tirer des conclusions exploitables.
